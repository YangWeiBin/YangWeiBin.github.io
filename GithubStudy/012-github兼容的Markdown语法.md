# github兼容的Markdown语法   
## 表格     
### Github不兼容写法  
1. 表头不空行   
|名称|时期|背景|特点|
|--|--|--|--|
|人工|管理|阶|段|



### Github兼容写法   
1. 正规写法：两个减号、两边必须|封口、表头前必须空行    

|名称|时期|背景|特点 |
|--|--|--|--|
|人工|管理|阶|段|

2. 一个减号   

|名称|时期|背景|特点|
|-|-|-|-|
|人工|管理|阶|段|
3. 表头不空行   
|名称|时期|背景|特点|
|--|--|--|--|
|人工|管理|阶|段|

4. 两边没用|封口   

名称|时期|背景|特点 
--|--|--|-- 
人工|管理|阶|段    


### 总结   
1. 表头前必须空行  
2. 每行换不换行均可
